## CPU 对应关系汇总

| GPU 线                                  | 最直接 CPU 对应 / 前身                                              | 关系判断                                                                                                 |
| -------------------------------------- | ------------------------------------------------------------ | ---------------------------------------------------------------------------------------------------- |
| **cuSZ → cuSZp → cuSZp2 → VGC/cuSZp3** | SZ / SZ3 / ZFP / FPZIP 等 CPU scientific lossy compressors    | GPU 线是为 HPC GPU workflow 重新设计 prediction / quantization / encoding pipeline；不是简单移植。                  |
| **cuSZ-i / cuSZ-Hi / PRISM**           | QoZ / HPEZ / SZ3 / MDR-MGARD progressive retrieval           | CPU 线偏高压缩率、interpolation、多级分解；GPU 线把 predictor、bitplane/lossless encoding、progressive retrieval 并行化。 |
| **ndzip-gpu**                          | ndzip, DCC 2021                                              | 最清楚的一对 CPU/GPU 版本：ndzip-gpu 是 ndzip 的 GPU parallelization。                                           |
| **FastLanesGPU**                       | FastLanes Compression Layout, PVLDB 2023                     | CPU/SIMD layout → GPU query processing。                                                              |
| **G-ALP**                              | ALP, SIGMOD/PACMMOD 2024                                     | CPU/vectorized floating-point database compression → GPU kernel-friendly decompression API。          |
| **ASPLOS 2025 Azami/Fallin/Burtscher** | 同一篇论文内置 CPU/GPU implementations                              | 不需要找对应论文；CPU/GPU 是同一套算法的两个实现。                                                                        |
| **GPULZ**                              | LZSS / Deflate 类 CPU 传统算法                                    | 更偏 generic GPU lossless compression，不是 FP 专用。                                                        |
| **lsCOMP**                             | 无明确 CPU 论文                                                   | 面向 light-source data 的 GPU single-kernel compressor。                                                 |
| **HP-MDR**                             | MDR / MGARD / multilevel decomposition progressive retrieval | CPU-centric progressive methods 的 GPU 版本/扩展。                                                         |

## 和你前面三条主线的对应

**FastLanes-like / 数据库列式 GPU 线：**

* Accelerating GPU Data Processing Using FastLanes Compression
* G-ALP: Rethinking Light-weight Encodings for GPUs

**Floating-point / scientific GPU 线：**

* ndzip-gpu
* Efficient Lossless Compression of Scientific Floating-Point Data on CPUs and GPUs
* cuSZ, cuSZp, cuSZp2, cuSZ-i, cuSZ-Hi, VGC/cuSZp3
* FZ-GPU, Aatrox, PRISM, HP-MDR
* lsCOMP，属于 domain-specific scientific instrument data compression

**Time-series GPU 线：**

目前主线里 **Chimp / Elf / Camel / DeXOR / Serf / AFC / ACTF / NPAC** 基本都是 CPU/streaming codec，没有看到明确 GPU 版本论文。它们的 GPU 化反而是一个明显空白：把 XOR/decimal-space/reference-selection 这类 FP time-series codec 做成 warp-friendly、branch-light、random-access-friendly GPU decompressor，应该是很自然的研究问题。
